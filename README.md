## 氣泡排序法 (Bubble Sort)

### 原理簡介
氣泡排序法是一種簡單的排序演算法。它重複地走訪過要排序的數列，一次比較兩個元素，如果它們的順序錯誤就把它們交換過來。走訪數列的工作是重複地進行直到沒有再需要交換，也就是該數列已經排序完成。這個演算法的名字由來是因為越小的元素會經由交換慢慢「浮」到數列的頂端。



### 複雜度分析
* **平均時間複雜度：** $O(n^2)$
* **最壞時間複雜度：** $O(n^2)$
* **最佳時間複雜度：** $O(n)$（當資料已排序時）
* **空間複雜度：** $O(1)$

## 選擇排序法 (Selection Sort)

### 原理簡介
選擇排序法的工作原理是：首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置，然後，再從剩餘未排序元素中繼續尋找最小（大）元素，然後放到已排序序列的末尾。以此類推，直到所有元素均排序完畢。



### 複雜度分析
* **平均時間複雜度：** $O(n^2)$
* **最壞時間複雜度：** $O(n^2)$
* **最佳時間複雜度：** $O(n^2)$
* **空間複雜度：** $O(1)$

---

## 插入排序法 (Insertion Sort)

### 原理簡介
插入排序法的工作原理是透過構建有序序列，對於未排序數據，在已排序序列中從後向前掃描，找到相應位置並插入。在實作上，通常採用 in-place 排序，因而在從後向前掃描過程中，需要反覆把已排序元素逐步向後挪位，為最新元素提供插入空間。



### 複雜度分析
* **平均時間複雜度：** $O(n^2)$
* **最壞時間複雜度：** $O(n^2)$
* **最佳時間複雜度：** $O(n)$（當資料幾乎有序時）
* **空間複雜度：** $O(1)$

---

## 合併排序法 (Merge Sort)

### 原理簡介
合併排序是建立在合併操作上的一種有效的排序演算法，該演算法是採用分治法（Divide and Conquer）的一個非常典型的應用。將已有序的子序列合併，得到完全有序的序列；即先使每個子序列有序，再使子序列段間有序。



### 複雜度分析
* **平均時間複雜度：** $O(n \log n)$
* **最壞時間複雜度：** $O(n \log n)$
* **最佳時間複雜度：** $O(n \log n)$
* **空間複雜度：** $O(n)$

---

## 快速排序法 (Quick Sort)

### 原理簡介
快速排序使用分治法策略來把一個串列分為兩個子串列。首先從數列中挑選一個元素作為「基準」（pivot），重新排序數列，所有元素比基準值小的擺放在基準前面，所有元素比基準值大的擺在基準的後面。之後再遞迴地把小於基準值元素的子數列和大於基準值元素的子數列排序。



### 複雜度分析
* **平均時間複雜度：** $O(n \log n)$
* **最壞時間複雜度：** $O(n^2)$（當基準選取不佳時）
* **最佳時間複雜度：** $O(n \log n)$
* **空間複雜度：** $O(\log n)$（遞迴堆疊）

## 學習心得

在撰寫這次「排序報告」與開發模擬頁面的過程中，我深刻體會到演算法效率對程式效能的影響。雖然基礎排序演算法（如氣泡排序）在邏輯上較易理解且實作簡單，但在處理大量數據時，其運算時間會劇烈增加。

相較之下，進階的分治演算法（如合併排序與快速排序）雖然實作邏輯較為複雜（涉及遞迴），但在實際應用中能大幅縮短處理時間。透過將這些演算法視覺化並上傳至 GitHub，我不僅掌握了演算法的底層原理，也學習了如何使用 Web 技術呈現抽象的資料結構運作過程，這對我未來在軟體開發與邏輯思考上都有很大的啟發。
