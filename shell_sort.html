希爾排序法 (Shell Sort)

一、運作原理

希爾排序法 是插入排序的改良版。插入排序在對「幾乎已排序」的資料操作時效率極高，但每次只能移動一個位置，效率受限。

希爾排序透過設定一個「增量（Gap）」，將原始陣列分割成數個子序列，分別對這些子序列進行插入排序。隨著演算法進行，增量會逐漸縮小，直到增量為 1 時，最後執行一次完整的插入排序。

二、運作過程圖解

範例陣列：[8, 9, 1, 7, 2, 3, 5, 4] (n=8)

階段 增量 (Gap) 分組與操作 結果
第一階段 gap = 4 比較 (8,2), (9,3), (1,5), (7,4) 並交換 [2, 3, 1, 4, 8, 9, 5, 7]
第二階段 gap = 2 比較距離為 2 的元素並進行插入排序 [1, 3, 2, 4, 5, 7, 8, 9]
第三階段 gap = 1 標準插入排序（此時陣列已基本有序） [1, 2, 3, 4, 5, 7, 8, 9]

*註：希爾排序顯著減少了長距離移動元素的次數。

三、演算法偽代碼 (Pseudocode)

procedure shellSort(A : list of items)
    n := length(A)
    gap := n / 2

    while gap > 0 do
        for i := gap to n - 1 do
            temp := A[i]
            j := i
            // 進行組內插入排序
            while j >= gap and A[j - gap] > temp do
                A[j] := A[j - gap]
                j := j - gap
            end while
            A[j] := temp
        end for
        gap := gap / 2 // 縮小增量
    end while
end procedure

四、複雜度分析

最佳時間複雜度

O(n log n)

平均時間複雜度

O(n1.3) ~ O(n²) (取決於增量序列)

最差時間複雜度

O(n²)

空間複雜度

O(1)

穩定性： 希爾排序是一種 不穩定排序 (Unstable Sort)。由於相同的元素可能會被分到不同的組別中進行插入排序，它們的相對順序可能會在排序過程中被改變。
