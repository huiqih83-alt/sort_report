選擇排序法 (Selection Sort)

一、運作原理

選擇排序法 的概念非常直觀。它將陣列分為「已排序」和「未排序」兩部分。

在每一輪中，演算法會在「未排序部分」中找到最小（或最大）的元素，然後將它與「未排序部分」的第一個元素交換位置。這樣一來，該元素就進入了「已排序部分」。

二、運作過程圖解

範例陣列：[64, 25, 12, 22, 11] (由小到大排序)

輪次 目前狀態 掃描結果 交換後結果
Pass 1 [64, 25, 12, 22, 11] 最小值是 11 [11, 25, 12, 22, 64]
Pass 2 [11, 25, 12, 22, 64] 最小值是 12 [11, 12, 25, 22, 64]
Pass 3 [11, 12, 25, 22, 64] 最小值是 22 [11, 12, 22, 25, 64]
Pass 4 [11, 12, 22, 25, 64] 最小值是 25 [11, 12, 22, 25, 64]

*註：底線部分代表「未排序區域」。

三、演算法偽代碼 (Pseudocode)

procedure selectionSort(A : list of items)
    n := length(A)
    for i := 0 to n-2 do
        // 假設目前未排序的第一個是最小的
        minIndex := i
        for j := i+1 to n-1 do
            if A[j] < A[minIndex] then
                // 發現更小的，記錄索引
                minIndex := j
            end if
        end for
        // 將找到的最小值與目前位置 i 交換
        if minIndex != i then
            swap(A[i], A[minIndex])
        end if
    end for
end procedure

四、複雜度分析

最佳時間複雜度

O(n²) (仍需掃描尋找極值)

平均時間複雜度

O(n²)

最差時間複雜度

O(n²)

空間複雜度

O(1) (原地排序)

穩定性： 選擇排序通常是 不穩定排序 (Unstable Sort)。因為在尋找極值並交換的過程中，可能會破壞相同數值元素原有的相對順序。

五、實驗分析

實驗環境

本實驗在相同的硬體環境下進行測試。測試資料為隨機亂數陣列，記錄排序所需的執行時間（毫秒），觀察排序時間隨資料規模 n 增加的變化趨勢。

性能分析觀察

時間變化趨勢： 當資料規模從 n = 1,000 增加到 n = 100,000 時（增加 100 倍），執行時間從 2.1 ms 增加到 21,148 ms（增加約 10,071 倍），完全符合 O(n²) 的時間複雜度曲線。

獨特特性： 選擇排序有個重要特性——無論輸入資料是否已排序，其時間複雜度始終為 O(n²)。因為該算法必須掃描整個陣列尋找極值。交換次數通常只有 n-1 次，這是所有 O(n²) 算法中最少的。

結論： 選擇排序在交換成本昂貴的場景下有優勢，但其穩定性差且不適合大規模資料。
