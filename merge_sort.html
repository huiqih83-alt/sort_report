合併排序法 (Merge Sort)

一、運作原理

合併排序法 採用了「分治法」(Divide and Conquer) 的策略。其運作分為兩個主要階段：

  1. 分割 (Divide)：將原始陣列不斷從中間對切，直到每個子陣列只剩下一個元素（此時每個子陣列都視為已排序）。
  2. 合併 (Merge)：將兩個已排序的子陣列合併成一個更大的已排序陣列，直到合併回原始大小。

二、運作過程圖解

範例：[38, 27, 43, 3]

[38, 27, 43, 3] (分割)
[38, 27] | [43, 3] (分割)
[38] | [27] | [43] | [3] (最底層)
[27, 38] | [3, 43] (第一次合併)
[3, 27, 38, 43] (最終合併完成)

三、演算法偽代碼 (Pseudocode)

合併排序通常使用遞迴實作：

procedure mergeSort(A : list of items)
    if length(A) <= 1 return A

    mid := length(A) / 2
    left := mergeSort(left half of A)
    right := mergeSort(right half of A)

    return merge(left, right)
end procedure

// 合併兩個有序陣列的邏輯
procedure merge(left, right)
    while left and right are not empty
        if left[0] <= right[0]
            append left[0] to result, remove from left
        else
            append right[0] to result, remove from right
    end while
    append remaining items to result
end procedure

四、複雜度分析

最佳時間複雜度

O(n log n)

平均時間複雜度

O(n log n)

最差時間複雜度

O(n log n)

空間複雜度

O(n) (需要額外空間存放合併結果)

穩定性： 合併排序是一種 穩定排序 (Stable Sort)。其最大的優點在於時間複雜度極其穩定，不受輸入資料排列順序影響。唯一的缺點是需要額外的記憶體空間。
